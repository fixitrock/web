## POS Route

The point-of-sale workflow lives at:

`/@username/pos`

This is where sellers can assemble an order and issue a receipt-ready transaction flow.

## Orders Route

Order management lives at:

`/@username/orders`

Use it to review and verify completed or in-progress transactions.

## Activity Views

The seller profile also exposes activity-oriented tabs powered by recent and top order data.

These views are useful for:

- Tracking recent order movement
- Seeing top-performing items
- Checking whether the counter flow is producing the expected records

## Recommended Counter Workflow

1. Open `/@username/pos`.
2. Search or select the required products.
3. Confirm customer details.
4. Finalize the order.
5. Open the receipt or order summary.
6. Recheck `/@username/orders` for the saved record.

## Operational Checks

- Prices should match the current catalog.
- Product quantities should reflect available stock if stock tracking is enabled.
- Receipt information should be complete before sharing with the customer.
- Recent activity should update after the order is saved.
