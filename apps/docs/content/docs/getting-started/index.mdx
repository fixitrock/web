Fixitrock mixes a public download portal with seller-specific storefronts.

## Core Patterns

- Public tools and file areas live under routes like `/space`, `/frp`, and `/space/Flash-Tool`.
- Seller storefronts use the `@username` pattern, for example `/@fixitrock`.
- Seller operation pages live under that username route, such as `/@username/products` and `/@username/pos`.
- Authentication starts from `/login`.

## Why This Structure Works

- Users can stay in the public content side without touching seller management pages.
- Sellers get a predictable route system under one username namespace.
- Support and onboarding get easier because route names match how the app is actually built.
