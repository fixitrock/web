## Seller Storefronts

Every seller storefront uses the `@username` route style.

Examples:

- `/@rockmobile`
- `/@fixitrock`

From the public storefront, users can:

- View the seller profile
- Open product tabs
- Browse categories
- Return to the seller home quickly

## Product Browsing

The public product experience is usually served through the seller route with a products tab or direct products page.

- Public view: `/@username?tab=products`
- Seller workspace view: `/@username/products`

If you are a customer, use the public tab route unless the seller has shared a private management URL with you for a specific reason.

## Receipts

Fix iT Rock also includes a receipt route for generated order paperwork.

Receipts are useful when:

- A seller finishes a POS order
- A customer needs a printable transaction summary
- The team wants to verify line items and totals

## Best Practice for Customers

1. Confirm the seller username before paying.
2. Review the product list or order summary carefully.
3. Save or share the receipt link after purchase.
4. Use the seller's support channel if any item details are missing.

## Seller-to-Customer Handoff

In practice, the cleanest flow is:

1. Seller creates the order.
2. Seller shares the receipt or order summary.
3. Customer validates items, price, and contact details.
4. Any correction is done by the seller in the seller workspace.
